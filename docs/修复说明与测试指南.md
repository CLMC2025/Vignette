# 微语单词｜修复说明与测试指南

> 目的：解决「开始学习 → 任务加载失败」的核心问题，并补齐离线兜底与 WebDAV-only 进化/同步能力。

---

## 1. 核心问题：开始学习后提示“任务加载失败”

### 1.1 复现条件（旧版本常见）
- 在首页点击「开始学习」进入学习页（ReadPage）。
- 当路由参数中携带 `queue`（队列）时，队列中的 `WordItem/FSRSState/ReviewHistory/WordDefinition` 等对象在跨页面传递后会变成 **普通对象**（prototype 丢失）。
- 学习页在后续逻辑里调用类方法（例如 `WordItem.clone()`、`FSRSState.toJSON()`、`WordDefinition.getContextMeaning()` 等），会触发运行时异常并进入兜底 Toast：`任务加载失败`。

### 1.2 根因定位
- **路由参数序列化导致 prototype 丢失**（本质上不是语法/编译问题，而是运行时数据形态变化）。
- 其次是：AI 请求失败/响应不符合预期时，错误边界不够明确，容易直接归结为“任务加载失败”。

### 1.3 修复方案
1) **路由参数归一化**（关键修复点）
- 新增：`entry/src/main/ets/utils/RouteParamNormalizer.ets`
- 在学习页初始化时对 `queue` 做归一化重建，确保类型与方法可用。

2) **学习页任务加载链路强化**
- 对 “取词 → 生成任务 → 渲染 UI” 全流程增加：
  - 统一错误分类（`ErrorClassifier`）
  - 加载状态反馈（loading）
  - 可恢复/可引导的错误弹窗

---

## 2. 离线兜底：AI 优先 → 离线模板兜底 → 最小故事兜底

### 2.1 目标
在 **无网络 / API 未配置 / API 超时 / 解析失败** 等情况下，仍然保证学习可继续，并且至少生成包含目标词的最小语境。

### 2.2 实现要点
- `DictionaryManager.generateVignetteForWord()` 内部流程：
  1. AI 成功：缓存 vignette（可控上限）。
  2. AI 失败：使用 `TemplateManager` 离线模板生成。
  3. 模板不足：生成最小 story（保证包含目标词）。

### 2.3 缓存策略（防止无限膨胀）
- 新增 `vignette_cache` 表（DB）
- 通过 `enableVignetteCache` 与 `vignetteCacheMaxBytes` 控制开关与容量（LRU + 上限）。

---

## 3. 可选增强：离线模板包（用户自行决定是否下载）

### 3.1 设计
- 默认只带少量内置模板，保证包体小。
- 用户可通过 WebDAV 下载一个 **加密的模板包**，导入后离线模板更丰富。

### 3.2 位置
- WebDAV 远端路径：`<remoteRoot>/templates/template_pack.json.enc`
- 本地：持久化到 AppStorage（key：`offline_template_pack_json`），由 `TemplateManager` 自动加载。

### 3.3 UI 入口
设置页 WebDAV 区块新增：
- 下载模板包
- 上传模板包
- 清除本地模板包

---

## 4. WebDAV-only 同步与自进化（无服务器）

### 4.1 同步（备份/恢复）
- 备份：导出本地数据 → AES 加密 → WebDAV PUT
- 恢复：WebDAV GET → 解密 → 导入本地（可在恢复前做 Snapshot 备份）

### 4.2 进化（Evolve）
- 上行：上传本地 review events（JSONL + AES）
- 下行：拉取其他设备的 events 合并
- 本地训练：在设备端用轻量优化器从 history 拟合更新策略（参考两篇论文的核心思路，做了工程化简化）
- 同步模型：上传/下载加密 policy 包

---

## 5. 测试方法

### 5.1 手动测试（必测）
1. **无网络**：
- 关闭网络/断开 Wi-Fi
- 首页点击「开始学习」→ 应进入学习页并生成任务（模板/最小故事兜底生效）

2. **API 未配置**：
- 设置页不填 API Key
- 首页点击「开始学习」→ 仍应可学（提示可引导配置 API）

3. **有网络 + API 正常**：
- 应优先走 AI vignette
- 重复进入同一词，命中缓存时速度明显提升

4. **WebDAV**：
- 配置坚果云/Nextcloud WebDAV
- 测试连接 → 备份 → 恢复（可在第二台设备验证）
- 上传复习日志 → 进化一次 → 下载模型（跨设备一致性）

### 5.2 单元测试（已补充）
- `RouteParamNormalizer.test.ets`：验证路由参数归一化不会丢失能力
- `TemplateManagerPack.test.ets`：验证外部模板包可导入/可清除
